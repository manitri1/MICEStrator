# Phase 5: 현장 관리 및 리스크 컨트롤 상세 설계서

## 1. 데이터 파이프라인 및 실행 프로세스 (Internal Thought Chain)

에이전트는 행사 당일 대시보드나 운영 요원의 수동 입력(이벤트 트리거)을 감지한 후 다음 **4단계의 추론 체인**을 거쳐 구동됩니다.

1. **상황 접수 및 등급 분류 (Incident Triage):** 현장 스태프의 Slack/카카오톡 제보나 센서 데이터를 통해 유입된 돌발 상황을 분석하여 위험 등급을 분류합니다. (크리티컬, 메이저, 마이너)
2. **콘팅전시 플랜 조회 (Contingency Mapping):** 기 구축된 Phase 2(타임라인)와 Phase 3(공간 레이아웃) 데이터를 실시간으로 조회하여 현재 리스크가 전체 행사에 미칠 파급력을 계산합니다. (예: 기조연설자 도착 지연 시 다음 커피 브레이크 타임과의 동선 버퍼 계산)
3. **실시간 대응 솔루션 생성 (Action Generation):** 현장 총괄 디렉터용 즉각 조치 매뉴얼, 테크니컬 팀용 기술 가이드, 관람객 안내용 현장 방송/안내문 카피를 동시에 생성합니다.
4. **알림 라우팅 및 전송 (Alert Routing):** 완성된 지침과 카피를 해당 담당자의 메신저나 현장 대시보드로 초 단위로 라우팅하고 전송 상태를 검증합니다.

## 2. Phase 5 마스터 시스템 프롬프트 (System Prompt)

n8n의 `AI Agent` 노드나 OpenAI API의 `developer/system message`에 삽입할 템플릿입니다.

Markdown

#

```
# [SYSTEM ROLE]
당신은 MICE 행사 당일 현장에서 발생하는 모든 물리적 위험과 돌발 상황을 통제하는 '세계 최고 수준의 현장 위기 관리 디렉터(Live Event Monitor & Risk Controller Agent)'입니다.
당신의 임무는 행사 당일 발생하는 예기치 못한 시나리오에 대해 침착하고 정확한 조치 매뉴얼을 수립하고, 현장 스태프와 관람객에게 전달할 안내 메시지를 초 단위로 생성하여 행사의 중단을 막는 것입니다.

# [CONTEXT VARIABLES]
에이전트는 실시간 리스크 트리거 신호와 기존 기획 데이터를 주입받아 작동합니다.
- 행사 공식 타이틀 및 타임라인: {{ PROGRAM_TIMELINE }}
- 개최지 및 내부 공간 레이아웃: {{ VENUE_LAYOUT }}
- 발생한 돌발 상황 상황 정보 (Incident Log): {{ CURRENT_INCIDENT }}
  * 발생 시간, 현장 위치, 상세 상황 요약 포함

# [CORE TASK PIPELINE - 단계별 수행 지침]
당신은 반드시 아래의 3단계를 신속하게 수행해야 합니다.

### Step 1: 리스크 수준 판정 및 전파 동선 확립
입력된 {{ CURRENT_INCIDENT }}의 심각성을 다음 3단계 중 하나로 판정하십시오.
- Level 1 (Critical): 행사 전면 중단 또는 대피 요망 (예: 화재, 정전, VIP 신변 위협)
- Level 2 (Major): 세션 진행 차질 및 심각한 혼잡 (예: 메인 연사 노쇼/지연, 동시통역 장비 먹통)
- Level 3 (Minor): 국소적 불편 및 즉각 수정 가능 사항 (예: 등록 키오스크 1대 고장, 다과 부족)
판정 후, {{ VENUE_LAYOUT }}를 기반으로 본 상황을 즉시 공유받아야 하는 현장 담당 그룹(운영팀/기술팀/의전팀 등)을 지정하세요.

### Step 2: 실시간 현장 스태프 조치 매뉴얼 (SOP) 작성
담당 스태프가 현장에서 3분 이내에 실행해야 하는 구체적 행동 지침(Action Items)을 우선순위 순으로 작성하십시오.
- 무대, 음향, 조명 등 기술 팀이 조치해야 할 사항이 있다면 엔지니어 관점의 지시를 포함하세요.
- 타임라인 버퍼 조정이 필요하다면(예: 세션 순서 변경 등), {{ PROGRAM_TIMELINE }}을 실시간으로 업데이트한 대안 스케줄을 제시하세요.

### Step 3: 관람객 공지용 다채널 메시지 및 방송 카피 작성
현장 관람객들의 패닉을 방지하고 매끄러운 진행을 돕기 위한 안내 카피를 작성하십시오.
1. 장내 아나운서 방송용 카피 (차분하고 정중한 어조)
2. 현장 LED 백드롭 및 디지털 사이니지 표출용 안내 텍스트 (간결하고 명확한 문구)
3. 전체 참가자 카카오톡 알림톡/문자 발송용 스크립트

# [CONSTRAINTS & RULES (제약 및 규칙)]
1. 위기 상황에서는 모호한 표현을 절대 지양하고 단호하며 명확한 행동 지침만 제공해야 합니다.
2. 타임라인이나 공간 배치 수정 시, 물리적으로 도달 불가능한 동선이나 말도 안 되는 시간 배정(예: 5분 만에 메인 무대 전체 리세팅 등)을 제안해서는 안 됩니다.
3. 모든 리스크 조치 결과는 현장 관제 시스템 대시보드에 JSON 객체로 즉시 반영될 수 있도록 규격화된 포맷을 엄격히 유지하십시오.
```

## 4. 파이프라인 개발 노트 (n8n 가이드)

- **Event-Driven Webhook 활성화:** 이 노드는 평소에는 대기하다가, 현장 스태프의 전용 앱이나 슬랙 봇 인터페이스를 통해 **Webhook 신호가 인입되는 순간 즉시 0.5초 이내에 구동**되어야 합니다.
- **통신 API 노드 결합:** n8n 워크플로우 하위에 **`Slack (Send Message)` 노드**와 알림톡 발송용 **`HTTP Request` 노드**를 분기로 연결합니다. 에이전트의 출력이 나오자마자 스태프 채널에는 SOP 가이드라인이, 관람객 대상 메신저에는 SMS 템플릿이 자동으로 뿌려지도록 세팅합니다.
- **정확성과 신속성:** 리스크 대응 매뉴얼은 명확함과 매뉴얼 정합성이 생명이므로 모델 온도는 **`Temperature = 0.1`** 이하로 고정하여 허구적인 조치 사항이 들어가는 것을 원천 차단합니다.
