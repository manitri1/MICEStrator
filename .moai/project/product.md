# MICEstrator — 제품 정의서

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | MICEstrator |
| **버전** | 0.1.0 (설계 단계) |
| **카테고리** | AI 에이전트 시스템 / MICE 업무 자동화 |
| **상태** | 신규 개발 |

## 제품 설명

MICEstrator는 **MICE(Meetings, Incentives, Conventions, Exhibitions) 행사 기획 전 과정을 AI 에이전트 파이프라인으로 자동화하는 시스템**입니다.

트렌드 조사(Phase 1)부터 행사 종료 후 ROI 분석(Phase 6)까지 6개의 전문 에이전트가 순차적으로 연결되어, 각 Phase의 출력이 다음 Phase의 입력으로 자동 상속됩니다. 기획자는 초기 정보(분야, 규모, 기간)만 입력하면 행사 기획에 필요한 모든 산출물(컨셉, WBS, 디자인 가이드, 연사 초청장, 마케팅 콘텐츠, ROI 보고서)을 자동으로 받을 수 있습니다.

## 핵심 문제 (Pain Points)

- MICE 기획자는 수십 개의 체크리스트와 문서화 작업을 반복적으로 수행
- 트렌드 조사 → 컨셉 도출 → 디자인 → 연사 섭외 → 마케팅 간 데이터 단절로 일관성 붕괴
- 행사마다 처음부터 다시 시작하는 비효율, 사후 ROI 측정 미흡

## 타깃 사용자

| 페르소나 | 설명 |
|---------|------|
| **MICE 전문 기획자** | 컨퍼런스·포상관광·컨벤션·전시를 기획하는 실무자. 반복 문서화 자동화 필요 |
| **컨벤션 뷰로 / 운영팀** | 지자체·협회 산하 MICE 운영 조직. 빠른 기획 초안과 ROI 보고 필요 |
| **행사 기획 스타트업** | 소규모 팀으로 대형 행사를 기획. 에이전트 자동화로 인력 한계 극복 |
| **기업 이벤트 담당자** | 사내 컨퍼런스·인센티브 투어 담당. 전문 지식 없이도 수준 높은 기획 필요 |

## 핵심 기능 (6-Phase 파이프라인)

### Phase 1 — 인텔리전스 토대 구축 (Trend & Strategy)
- **입력:** 희망 분야/산업, 준비 기간, 개최 규모
- **출력:** 국문/영문 행사명, 슬로건, 부제, 기획 배경, 타깃 페르소나 프로파일
- **핵심:** PEST 분석 기반 메가트렌드 → 차별화된 컨셉 도출. 전체 프로젝트의 **Single Source of Truth** 생성

### Phase 2 — 프로젝트 아키텍처 수립 (WBS & Timeline)
- **입력:** Phase 1 기획 문서, 가용 인력 수, 총 준비 기간
- **출력:** WBS 기반 역할 분담표, 일자별 마일스톤 테이블
- **핵심:** PM/홍보/연사섭외/운영/테크 부서 의존성 고려한 역산 스케줄링

### Phase 3 — 비주얼 아이덴티티 구체화 (Visual Identity)
- **입력:** Phase 1 컨셉/슬로건/키워드
- **출력:** 메인 컬러(Hex 3종), 디자인 무드, Canva/Midjourney용 이미지 생성 프롬프트
- **핵심:** 이후 모든 Phase에 자동 주입되는 **브랜드 컨셉 메모리** 확립

### Phase 4 — 연사 소싱 & 자동화 아웃리치 (Speaker Sourcing)
- **입력:** 세부 세션 구성, 연사 조건/예산 범위
- **출력:** 연사 후보군 타깃팅 리스트, 선정 사유, 커스텀 초청 이메일 템플릿
- **핵심:** 세션 기획 의도와 연사 시너지를 강조한 비즈니스 격식 이메일 자동 생성

### Phase 5 — 디지털 에셋 & 마케팅 엔진 (Digital Assets & Marketing)
- **입력:** 확정 연사 라인업, 세션 정보, Phase 3 브랜드 가이드
- **출력:** SNS 티저 스크립트(인스타·링크드인 분리), 홈페이지 레이아웃 구조, 오프닝 음악 생성 프롬프트
- **핵심:** 타깃 청중 성향(Z세대 vs 직장인)에 따른 채널별 맞춤 콘텐츠

### Phase 6 — 사후 관리 & 효과 측정 (Post-Event ROI Analysis)
- **입력:** 참가자 데이터, 만족도 설문 원본, 비즈니스 매칭 성과 데이터
- **출력:** KPI 대시보드 JSON, 설문 텍스트 마이닝 결과, 차기 행사 제언 리포트
- **핵심:** 데이터 왜곡 없는 객관적 분석, 차기 행사 액션 플랜 자동 도출

## 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| **Single Source of Truth** | Phase 1 출력이 전체 프로젝트 마스터 데이터. 이후 모든 Phase가 이를 참조 |
| **컨셉 메모리 자동 주입** | Phase 3에서 확정된 브랜드 컨셉(컬러, 폰트, 무드)이 Phase 4·5에 자동 연동 |
| **JSON 스키마 준수** | 모든 에이전트 출력이 정해진 JSON 스키마로 구조화 (대시보드/문서 자동 생성 가능) |
| **컴포넌트 기반 출력** | 각 Phase 출력이 독립된 컴포넌트로 분리 (재사용 및 다음 Phase 입력 가능) |
| **데이터 무왜곡 원칙** | Phase 6 분석 시 긍정·부정 피드백 모두 가감 없이 보고 |

## 결과 편집 & 멀티모달 리뷰 (SPEC-CHAT-001)

각 Phase 결과를 AI 채팅으로 부분 수정하고, 화면을 캡처해 멀티모달 피드백을 받을 수 있는 기능.

| 기능 | 설명 |
|------|------|
| **PhaseChat 채팅 패널** | 자연어로 결과 수정 요청 → AI가 변경 diff 제안 → 적용/취소 |
| **ScreenshotCapture** | html2canvas로 현재 화면 캡처 → GPT-4o Vision 멀티모달 리뷰 |
| **PUT /api/phase-result** | 객체 배열 인덱스 병합 deepMerge — 나머지 필드 보존 |
| **POST /api/chat/phase-edit** | 스트리밍 편집 API (이미지 첨부 지원) |
| **useParams 하이드레이션 수정** | Phase 1~6 페이지 `use(params)` → `useParams()` — React 이벤트 버그 수정 |

## Phase 스테일 감지 & 재생성 워크플로우 (SPEC-UI-001)

상위 Phase가 갱신되면 하위 Phase에 스테일 배너를 표시하고, 원클릭 재생성으로 최신 데이터로 갱신하는 UX.

| 기능 | 설명 |
|------|------|
| **PhaseStaleBanner** | 상위 Phase 갱신 감지 시 "구버전 기반" 경고 배너 표시 |
| **원클릭 재생성** | 배너의 "재생성" 버튼 클릭 → 폼 자동 제출 (`formRef.requestSubmit()`) |
| **재생성 성공 메시지** | 재생성 완료 후 초록색 성공 메시지 표시 |
| **reason prop** | `chat-edit` (채팅 편집 후) / `phase-rerun` (Phase 재실행 후) 원인 구분 |

## 이벤트 요약 대시보드 (SPEC-SUMMARY-001)

홈 대시보드에 Phase 1~4 결과를 집계해 행사 핵심 정보를 한눈에 표시하는 기능.

| 기능 | 설명 |
|------|------|
| **EventSummaryBanner** | 행사명, 컨셉, 주요 연사, 마케팅 키워드를 홈에 배너로 표시 |
| **GET /api/event-summary** | Phase 1~4 phaseResults를 집계해 EventSummary 타입으로 반환 |
| **실시간 집계** | 최신 Phase 결과 기반 동적 업데이트 |

## Canvas 랜딩페이지 프롬프트 (Phase 5)

Phase 5 랜딩페이지 탭 내에 Gemini Canvas용 프롬프트 생성 기능이 인라인으로 통합됨.

| 기능 | 설명 |
|------|------|
| **인라인 프롬프트 생성** | 랜딩페이지 탭 하단에서 바로 Canvas 프롬프트 생성 |
| **복사 버튼** | 생성된 프롬프트를 한 번에 전체 복사 |
| **재생성 지원** | 이미 생성된 프롬프트를 다시 생성 가능 |

## 비범위 (Out of Scope)

- 실제 행사 현장 운영 시스템 (참가자 체크인, 현장 앱)
- 결제·티켓 판매 플랫폼 연동
- 실시간 라이브 스트리밍 솔루션
- 참가자 CRM 데이터베이스 관리
